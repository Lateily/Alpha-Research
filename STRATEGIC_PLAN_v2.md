## 一、先批判自己的第一稿思路

我最初的本能是列一个"五周功能清单"：第一周加这个，第二周加那个。这个思路是错的。

**错在哪里：**

功能堆砌不等于alpha能力提升。我们现在的平台已经有了：DeepResearch三遍扫描、Barra-lite宇宙评分、CVD信号、摆动信号、反向DCF、宏观压测、回测引擎、纸上交易、Supabase快照、Telegram推送。但它们是**孤岛**——每个模块单独运行，之间没有决策流。

一个能稳定盈利的交易系统不是功能的总和，而是**一个首尾相接的闭环**：信号 → 决策 → 仓位 → 跟踪 → 反馈 → 信号校准。

我们现在的问题不是缺功能，而是缺闭环。

## 二、对"终极目标"的诚实解构

你说的目标："买方PM分析思维 + 量化技术面 + 实时交易信息 + 每日交易策略 + Portfolio构建 + 稳定收益"

这里有一个根本性张力需要正视：

**买方PM思维**和**量化技术面**在时间维度上是对立的。

- PM思维：持有周期3-18个月，基于基本面错误定价，容忍波动
- 量化技术面：持有周期1-20天，基于价格/流量信号，追求时机精确

**很多平台试图同时做这两件事，结果哪件都做不好。**

第二轮批判后，我的结论是：这两者不是对立的，而是**层级关系**。

```
战略层（月频）：PM思维 → 确定"持有什么"
战术层（日频）：量化技术 → 确定"何时进出"
```

这就是顶级对冲基金的实际运作方式：基本面分析师告诉你买什么，量化系统告诉你什么时候按下扳机。我们的平台应该复刻这个层级，而不是把两者混在同一个决策里。

## 三、现状真实诊断（不美化）

### 3.1 现有架构的实际状态

```
数据层        ✅ 基本稳定（Eastmoney, yfinance, 资金流向）
研究层        ✅ 成熟（DeepResearch三遍 + 对抗性框架 + 实时基本面注入）
信号层        ✅ 完整（MA/RSI/MACD/KDJ/BB + CVD AS模型）
Barra评分     ✅ 已上线（五因子宇宙排名）
纸上交易      ⚠️ 存在但断层（有P&L但无信号驱动）
回测          ⚠️ 有引擎但数据点不足（< 60天历史 → 显示insufficient_data）
Portfolio     ❌ 不存在（100%仓位单票，无构建逻辑）
信号→交易闭环 ❌ 完全断开（信号文件存在，paper_trading.py从不读它）
P&L归因       ❌ 不存在（无法知道是哪个信号赚的钱）
VP Score      ❌ 仍是硬编码（5只票的VP是手工填的静态数字）
```

### 3.2 最严重的三个结构性缺陷

**缺陷一：决策层缺失**

我们有数据，有信号，有研究——但没有一个层把它们合成为"今天的交易决策"。每天早上系统推送Morning Report，但用户需要自己手动判断信息 → 这意味着那80%自动化的承诺没有实现。

**缺陷二：反馈回路不存在**

我们的VP Score是主观起点，但从未被市场数据校准过。我们的摆动信号不知道自己的历史胜率。回测引擎存在，但它回测的VP Score是硬编码的假数字。整个系统在一个开环中运行，永远无法自我改进。

**缺陷三：仓位管理是空白**

现在的"portfolio"是：一只票，100%仓位。这不是portfolio，这是单票押注。没有相关性管理，没有波动率加权，没有最大回撤控制。

## 四、目标架构：三层决策引擎

（第三轮修订后的核心架构设计）

```
┌─────────────────────────────────────────────────────────────────┐
│  LAYER 3 — 战略层（周频回顾）                                      │
│  输入：宏观政策信号 + 板块资金流向 + VP Score                        │
│  输出：持仓宇宙（哪些票可以持有）+ 方向性偏向（多/空/现金）             │
│  工具：regime_config + DeepResearch + 对抗性研究框架               │
└──────────────────────────┬──────────────────────────────────────┘
                           │ 传递：approved_universe[]
┌──────────────────────────▼──────────────────────────────────────┐
│  LAYER 2 — 战术层（日频执行）                                      │
│  输入：approved_universe + 信号文件 + 资金流向 + 昨日持仓           │
│  输出：今日交易建议（进场/加仓/减仓/观望）+ 信号置信度评分            │
│  工具：swing_signals + CVD + Barra alpha + 信号置信度合成器        │
└──────────────────────────┬──────────────────────────────────────┘
                           │ 传递：trade_signals[]
┌──────────────────────────▼──────────────────────────────────────┐
│  LAYER 1 — 执行层（实时跟踪）                                      │
│  输入：trade_signals + 持仓 + 实时价格                             │
│  输出：仓位状态 + P&L归因 + wrongIf监控 + Telegram推送              │
│  工具：paper_trading + positions + prediction_log                 │
└─────────────────────────────────────────────────────────────────┘
```

**关键洞见：** Layer 3告诉你"可以买什么"，Layer 2告诉你"现在该买还是等"，Layer 1告诉你"买了之后怎么管"。三层各自有独立逻辑，但通过标准化数据接口相互通信。

### Week 1（2026-04-21至04-27）：打通决策闭环

**这一周的唯一目标：让信号能够驱动纸上交易决策。**

目前状态：信号存在（signals_300308_SZ.json），交易存在（trades.json），两者之间没有桥梁。
目标状态：每天GitHub Actions运行后，自动生成一份"今日交易建议"文件。

#### Task 1A：信号置信度合成器（`scripts/signal_confluence.py`）

这是Layer 2的核心。不是单独看某一个信号，而是计算当前所有信号的综合置信度：

```python
# 信号权重矩阵（基于信号类型和强度）
SIGNAL_WEIGHTS = {
    'MA_CROSS_BULLISH':      {'weight': 3, 'time_decay_days': 10},
    'MA_BOUNCE_BULLISH':     {'weight': 2, 'time_decay_days': 5},
    'RSI_OVERSOLD':          {'weight': 2, 'time_decay_days': 3},
    'VOLUME_BREAKOUT':       {'weight': 3, 'time_decay_days': 2},
    'CVD_FLOOR_FORMED':      {'weight': 4, 'time_decay_days': 5},  # 最高权重：资金面确认
    'CVD_BEARISH_DIVERGENCE':{'weight': -4, 'time_decay_days': 3},
    'RSI_OVERBOUGHT':        {'weight': -2, 'time_decay_days': 3},
    'MA_DEATH_CROSS':        {'weight': -3, 'time_decay_days': 10},
}

# 输出：每票的综合置信度得分（-100到+100）
# +60以上：考虑进场
# -60以下：考虑离场
# 中间：观望
```

同时加入**信号独立性检验**：MA多头排列和MACD金叉通常同时出现（高度相关），不应该被计为两个独立信号。合成器需要识别信号族群，只对真正独立的信号给予叠加权重。

### Week 2（2026-04-28至05-04）：Portfolio构建基础

**这一周的唯一目标：从单票押注进化到真正的Portfolio。**

目前的portfolio是：一只票，100%仓位。这是致命的。不管分析多精准，单票押注的风险回报比都是非理性的。

#### Task 2A：仓位规模计算器

基于两个维度确定仓位大小：
1. **VP Score权重**：VP越高，分配越多（但有上限）
2. **波动率调整**：ATR越高，仓位越小（等波动率暴露，不是等名义金额）

```python
def compute_position_size(ticker, vp_score, atr_pct, portfolio_value, n_positions=5):
    """
    Base allocation: portfolio_value / n_positions  (等权基准)
    VP adjustment:   ×(0.7 + 0.6 × (vp_score - 50) / 50)  (VP50→×0.7, VP80→×1.06)
    Vol adjustment:  × target_atr_pct / atr_pct             (高波动压仓位)
    Cap:             max 30% single position
    Floor:           min 5% single position
    """
```

#### Task 2B：相关性检查器

在添加新仓位时，检查与现有仓位的相关性。如果两只票的30天收益率相关系数 > 0.7，系统应该警告"高相关性：持有两者等于持有同一风险"。

这个检查不需要复杂的数据——用`public/data/ohlc_*.json`里已有的收益率数据就够。

#### Task 2C：行业敞口警报

当同一Shenwan行业的持仓比例 > 50%时，自动在DailyDecision里标红。当前100%电子行业是明显的集中风险。

**Week 2结束时的验收标准：** 新建一个模拟组合，5只票，仓位经过波动率调整，各票权重合理。

当仓位关闭时，更新exit_signal和pnl_pct。

#### Task 4B：信号质量看板

每月运行一次统计：每种信号类型的历史胜率和平均盈亏比。

```
CVD_FLOOR_FORMED:      5次 | 胜率60% | 平均盈利+8.2% | 平均亏损-3.1% | 期望值+3.8%
MA_CROSS_BULLISH:      8次 | 胜率50% | 平均盈利+6.1% | 平均亏损-4.2% | 期望值+0.95%
RSI_OVERSOLD alone:    3次 | 胜率33% | 平均盈利+4.0% | 平均亏损-5.5% | 期望值-2.4%
```

这个统计会驱动信号权重调整——如果RSI_OVERSOLD单独出现的胜率只有33%，它的权重应该被降低（或要求必须有其他信号配合才触发）。

#### Task 4C：wrongIf自动监控

Prediction Log里每条研究都有wrongIf字段（格式已经升级为"指标+阈值+动作"）。GitHub Actions每日运行时，解析每条wrongIf条件，检查是否触发，是则推送Telegram：

## 六、不要建什么（同等重要）

这是第三轮批判后最重要的结论。

**❌ 不要现在建ML预测模型**
没有3个月以上的干净信号→收益数据，任何ML模型都是对噪声的过拟合。先积累数据，6个月后再考虑。

**❌ 不要追求实时行情（tick级别）**
对于持有周期3-20天的策略，T+0日内快照完全够用。追求秒级数据需要付费API（月费$50-200）和复杂的WebSocket架构，性价比极低。

**❌ 不要现在做港股做空**
做空需要融券，纸上交易模拟做空有价值，但在系统未经验证之前不要在真实资金上实施。

**❌ 不要增加更多AI研究遍数**
研究质量边际回报递减。Pass 3不会比Pass 2好多少，但成本翻倍。现在的瓶颈是闭环，不是研究深度。

**❌ 不要同时建太多仓位**
5只票是上限。10只票看起来分散，实际上超出了单人PM的有效监控能力。深度优于广度。

## 七、"稳定收益"的诚实预期

这是我第三轮批判自己时最需要直面的问题。

**"5周内实现稳定收益"是不现实的目标**，原因：

1. **信号未经校准**：CVD、MA信号的历史胜率未知。需要至少50-100个信号→结果的数据点才能做统计结论，按每月15个信号计，需要3-7个月。

2. **样本期太短**：我们的旭创仓位持有7天，赚了15.47%。这是alpha还是运气？市场在这7天整体涨了多少？不知道。

3. **回测数据不足**：`backtest.py`现在显示`insufficient_data`，因为历史数据不到60天。

**5周内可以实现的诚实目标：**

- ✅ 一个首尾相接的决策闭环（信号→建议→仓位→归因→校准）
- ✅ 一个至少有5只票的模拟组合，仓位经过波动率调整
- ✅ 每日晨会推送，包含可操作的交易建议
- ✅ 信号质量追踪开始积累（为3个月后的校准做准备）
- ✅ wrongIf自动监控，实时捕捉thesis失效信号

**3个月后（7月底）才能合理评估：**
- 信号胜率统计（需要50+样本）
- 回测有意义的历史数据（需要60+天）
- Brinson-Fachler归因（需要真实P&L序列）

**6个月后才值得考虑：**
- 真实资金（小仓位）
- ML信号增强
- 做空组合

## 十、致自己的一句话

我在写这份文档时批判过三次自己的思路。每一次批判，结论都更清晰：

**这个平台成功的关键不是更多功能，而是现有功能之间的诚实对话。**

信号要能驱动决策，决策要能被归因，归因要能校准信号。

这个闭环一旦建立，剩下的所有优化——ML增强、港股做空、期权流分析——都有了生长的土壤。

在闭环建立之前，任何新功能都只是把孤岛变大，而不是把孤岛连成大陆。
