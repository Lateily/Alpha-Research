# PR-B — 决策书 vs 普通 AI 摘要 基线对照(2026-06-13)

| | |
|---|---|
| 问题 | Core Thesis 的决策书流程,相对一份"普通 AI 摘要",有没有**可测量的信息增量**?还是只是高级文书? |
| **不回答** | 决策书的论点最终**对不对**(那是 2026-07-10 起的前向 checkpoint)。PR-B 测的是**研究质量/信息增量**,不是 forward 正确性。 |
| 方法 | 盲生成基线 + 盲打分(实现 STRATEGY_LAB_AGENT.md 的角色隔离:把 T1 的手从基线和打分上都拿开) |
| 样本 | n=2(华海清科 688120.SH、鼎龙股份 300054.SZ);**case-level,不是统计证明** |
| 结论 | **决策书流程产生真实、可定位的信息增量:平均 +26.2 分,集中在可证伪性(+38/+42)与估值纪律(+26/+28)。它不是高级文书。** 但这是 alpha 主张的**必要不充分**条件——前向 checkpoint 是第二道门。 |

## 1. 方法(为什么可信)

这是 PR-B 最容易自欺的地方:如果 T1 自己又写决策书、又写基线、又打分,结果不可信(违反 v1.2 宪法第 4 条:同一上下文不能自我批准)。所以:

1. **盲生成基线**:两个全新 web-enabled agent 各写一份华海/鼎龙的"普通投资摘要"——它们**不知道这是要被对比的基线**,被要求"尽量写好"(防稻草人)。两份基线都得出了和我们方向相近的结论("优质公司、不便宜的价格"),证明它们是诚实的 steelman。
2. **匿名化**:决策书剥掉 provenance 头(model-pilot / 红队 / content-lock 等身份标识),只留分析内容;每只票随机标 甲/乙(华海 甲=摘要乙=决策书,鼎龙翻转 甲=决策书乙=摘要),位置不泄露类型。
3. **盲打分**:第三个全新 agent 拿五轴量规(`FIVE_AXIS_RUBRIC.md`)给四份匿名材料逐轴打分、逐轴引证据,**不知道哪份是"我们的"**,且被明确要求"不奖励格式/长度"。

## 2. 结果(按文档类型解码)

| | 普通 AI 摘要 | 决策书 | 增量 |
|---|---|---|---|
| 华海清科 | 64.4 | 88.4 | **+24.0** |
| 鼎龙股份 | 59.4 | 87.8 | **+28.4** |
| **平均** | **61.9** | **88.1** | **+26.2** |

**逐轴拆解(增量集中在哪)**:

| 轴 | 华海增量 | 鼎龙增量 | 读法 |
|---|---|---|---|
| 1 具体性 | +18 | +20 | 中等——散文也有股价/财务数字,扣分在结论端缺入场价/盈亏比/仓位档 |
| **2 可证伪性** | **+38** | **+42** | **最大**——决策书的机制化 wrong-if(指标+阈值+来源+日期)是散文最难自然满足、结构最受益的一轴 |
| **3 估值纪律** | **+26** | **+28** | **第二**——熊/基/牛三场景+模型锚+盈亏比纪律 vs 散文的"PE 偏高"一句话 |
| 4 信息增量 | +20 | +30 | variant perception 拆分(华海"激励底线 10.9亿 vs 卖方 14亿 +28% 缺口"最尖锐)vs 共识复述 |
| 5 风险识别 | +18 | +22 | 中等——散文也列了主要风险,扣分在偏笼统、缺流动性/拥挤度维度 |

## 3. 一个意外的交叉验证(量规校准)

盲打分 agent **不知道哪份是我们的**,却给两份决策书打了 88.4 / 87.8;Junyan 的人工五轴红队此前给的是 **84.4 / 82.6**。两个独立、互盲的打分者对决策书质量的判断高度吻合(差 ~4-5 分,盲打分略高)——这说明:① 五轴量规是可校准、可复现的,不是主观随意;② 决策书的质量不依赖"谁来打分"。

## 4. 诚实边界(这个结果能说什么、不能说什么)

**能说**:
- 决策书流程相对零成本替代(普通 AI 摘要)有**真实、可定位的信息增量**(+26.2),且增量在它**被设计来强制的轴**上(可证伪性、估值纪律)最大——这正是流程价值的证据,不是格式偏见。
- 它**不是高级文书**:如果决策书只是把摘要排成表格,可证伪性和估值纪律不会差 +38/+28。

**不能说**:
- **不是 alpha 证明**。PR-B 测"写得空不空",不测"猜得对不对"。决策书可以写得很可证伪,但论点照样可能错——那由 7/10 起的前向 checkpoint 裁。PR-B 是 alpha 主张的**第一道、也是最便宜的门(必要不充分)**;前向是第二道门。
- **n=2,case-level**,不是统计显著。
- **auto 盲打分是 floor**,Junyan 的人工红队仍是 measure of record(§3 的吻合只是给了量规可信度,不替代人工裁决)。
- 基线是 **AI 生成的 steelman**,不是"市面上最好的人类研报"——增量是相对"普通 AI 摘要"这个具体基准,不是相对全市场最佳。

## 5. 这一步在 alpha 阶梯上的位置

回答"Core Thesis 是不是 alpha"需要三级证据,按顺序:

1. **PR-B:打得过普通 AI 摘要基线吗?(信息增量)** ← **本文,YES +26.2**
2. **前向 checkpoint:论点按预测兑现吗?(track record)** ← 2026-07-10 起,n 慢慢长
3. 最终:足够多论点的前向记录 + 真金归因,边际可归因于流程、扣成本后还在

**我们现在过了第 1 级。** Core Thesis = primary alpha-**hypothesis** engine 得到了第一个量化支撑;它是不是 alpha,仍由前向法庭裁。
